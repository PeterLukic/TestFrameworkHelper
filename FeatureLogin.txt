Feature: Login

  Background:
    Given The OrangeHRM login page is open

  Scenario: Successful Login with Valid Credentials
    Given The Username field is "Admin"
    And The Password field is "admin123"
    When I click the "Login" button
    Then I should be redirected to the dashboard

  Scenario: Login with Empty Username
    Given The Username field is ""
    And The Password field is "admin123"
    When I click the "Login" button
    Then I should see an error message "Username is required"

  Scenario: Login with Empty Password
    Given The Username field is "Admin"
    And The Password field is ""
    When I click the "Login" button
    Then I should see an error message "Password is required"

  Scenario: Login with Empty Username and Password
    Given The Username field is ""
    And The Password field is ""
    When I click the "Login" button
    Then I should see an error message "Username is required"
    And I should see an error message "Password is required"

  Scenario: Login with Invalid Username and Valid Password
    Given The Username field is "InvalidUser"
    And The Password field is "admin123"
    When I click the "Login" button
    Then I should see an error message "Invalid username or password"

  Scenario: Login with Valid Username and Invalid Password
    Given The Username field is "Admin"
    And The Password field is "wrongPassword"
    When I click the "Login" button
    Then I should see an error message "Invalid username or password"

  Scenario: Login with Invalid Username and Invalid Password
    Given The Username field is "InvalidUser"
    And The Password field is "wrongPassword"
    When I click the "Login" button
    Then I should see an error message "Invalid username or password"

  Scenario: Username Field Auto-Focus
    Given The OrangeHRM login page is open
    Then The Username field should have focus

  Scenario: Password Field Type is Masked
    Given The OrangeHRM login page is open
    When I enter "admin123" into the Password field
    Then The Password field should display masked characters (e.g., asterisks or dots)

  Scenario: Verify "Forgot your password?" link presence
    Given The OrangeHRM login page is open
    Then I should see a link with the text "Forgot your password?"

  Scenario: Verify Social Media Links
    Given The OrangeHRM login page is open
    When I click the LinkedIn link
    Then I should be redirected to the OrangeHRM LinkedIn page
    And I should see the URL contains "linkedin.com/company/orangehrm"

    When I click the Facebook link
    Then I should be redirected to the OrangeHRM Facebook page
    And I should see the URL contains "facebook.com/OrangeHRM"

    When I click the Twitter link
    Then I should be redirected to the OrangeHRM Twitter page
    And I should see the URL contains "twitter.com/@orangehrm"

    When I click the YouTube link
    Then I should be redirected to the OrangeHRM YouTube page
    And I should see the URL contains "youtube.com/OrangeHRMInc"

  Scenario: Login with Username Containing Special Characters
    Given The Username field is "Admin!@#$"
    And The Password field is "admin123"
    When I click the "Login" button
    Then I should either be logged in successfully or see an appropriate error message. #Needs further investigation of allowed characters.   

  Scenario: Login with Password Containing Special Characters
    Given The Username field is "Admin"
    And The Password field is "admin123!@#"
    When I click the "Login" button
    Then I should either be logged in successfully or see an appropriate error message. #Needs further investigation of allowed characters.   

  Scenario:  Login with Username exceeding maximum allowed length
    Given The Username field is a very long string exceeding the allowed length
    And The Password field is "admin123"
    When I click the "Login" button
    Then I should either be logged in successfully or see an appropriate error message. #Needs further investigation of allowed characters.   

  Scenario:  Login with Password exceeding maximum allowed length
    Given The Username field is "Admin"
    And The Password field is a very long string exceeding the allowed length
    When I click the "Login" button
    Then I should either be logged in successfully or see an appropriate error message. #Needs further investigation of allowed characters.   

  Scenario: Verify CSRF Token is present in Login Form
    Given The OrangeHRM login page is open
    Then I should see a CSRF token field in the login form

  Scenario: JavaScript Disabled
    Given JavaScript is disabled in my browser
    When I attempt to login with valid credentials
    Then The login should fail and display a relevant error message or prevent login.

  Scenario: Verify Color Scheme
    Given The OrangeHRM login page is open
    Then The primary action button should have a background color of #e27304
    And The primary font color should be white (#FFFFFF)
    And The background should have a gradient from yellow to orange (#f6f661 to #F35C17)
